[문제 링크](https://www.acmicpc.net/problem/11053)

```c++
#include <bits/stdc++.h>
using namespace std;

int N;
int arr[1005];
int D[1005];

int main(){
  ios::sync_with_stdio(0);
  cin.tie(0);

  cin >> N;
  for(int i=1; i<=N; i++){
    cin >> arr[i];
    D[i] = arr[i];
  }

  for(int i=1; i<=N; i++){
    for(int j=1; j<=i; j++){
      if(arr[j] < arr[i]) D[i] = max(D[i], D[j]+arr[i]);
    }
  }
  cout << *max_element(D, D+N+1);
}
```

## 1. DP 점화식 정의

`D[i]` : i번째 숫자 `arr[i]`를 마지막으로 포함하는 증가 부분 수열 중 합의 최댓값.

## 2. 점화식의 원리: "내 앞에 올 수 있는 부분 수열 합 중 누가 제일 큰가?"

내가 `arr[i]`라면, 내 앞(j < i)에 있는 친구들 중에서 나보다 작은(arr[j] < arr[i]) 친구들을 훑어본다.

1. 조건 확인: `if(arr[j] < arr[i]`
  - 나보다 작은 애들 뒤에는 내가 붙을 수 있음

2. 최댓값 갱신: `D[i] = max(D[i], D[j]+arr[i])`
  - 지금까지 내가 알고 있던 합보다 어떤 수열 뒤에 나를 붙인 합이 더 크다면? 
  -> 그 수열 뒤에 붙는걸로 갈아탄다

## 3. 시뮬레이션
입력이 `1, 100, 2, 50` 이라고 가정

① 첫 번째 숫자: [1]
- 앞에 아무도 없음
- `D[1] = 1` (나 혼자 파티니까 합은 1)

② 두 번째 숫자: [100]
- 앞에 있는 1을 본다. 1은 100보다 작음 (OK)
- 1의 뒤에 서면? `D[1] + 100 = 101`
- `D[2] = 101`

③ 세 번째 숫자: [2]
- 앞에 있는 1을 본다. 1은 2보다 작음 (OK)
- 1의 뒤에 서면? `D[1] + 2 = 3` -> 일단 `D[3] = 3` 저장.
- 앞에 있는 100을 본다. 100은 2보다 작을까? (No)
- 최종 `D[3] = 3`

④ 네 번째 숫자: [50] (★가장 중요!)
- 이제 50은 자기 앞의 1, 100, 2를 다 검사한다.
- j=1 (숫자 1): 1은 50보다 작음. `D[1] + 50 = 51.` (현재 최댓값: 51)
- j=2 (숫자 100): 100은 50보다 큼. 못 붙음.
- j=3 (숫자 2): 2는 50보다 작음. D[3]은 아까 3이었음 D[3] + 50 = 53.
- 결정: 51보다 53이 더 크네? 2번 숫자 뒤에 붙는 게 이득이다!
- 최종 `D[4] = 53`
