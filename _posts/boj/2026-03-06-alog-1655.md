[문제 링크](https://www.acmicpc.net/problem/1655)

```c++
#include <bits/stdc++.h>
using namespace std;

int main(){
  ios::sync_with_stdio(0);
  cin.tie(0);

  int N, t;
  priority_queue<int> maxHeap;
  priority_queue<int, vector<int>, greater<>> minHeap;

  cin >> N;
  for(int i=0; i<N; i++){
    cin >> t;
    // 두 힙 사이즈가 같을땐 왼쪽(maxheap)그룹에 넣어서 중간값이 홀수던 짝수던 작은값을 뽑아냄
    if(maxHeap.size() <= minHeap.size()) maxHeap.push(t);
    else minHeap.push(t);
    if(!minHeap.empty() && minHeap.top() < maxHeap.top()){
      minHeap.push(maxHeap.top()); maxHeap.pop();
      maxHeap.push(minHeap.top()); minHeap.pop();
    }
    cout << maxHeap.top() << '\n';
  }
}
```

## 핵심 아이디어: 데이터를 반으로 가른다!!!

처음엔 `엥 왜 이 문제가 힙이랑 무슨 상관??`했는데 풀이를 보고 이마를 탁쳤따.

들어오는 숫자들을 일렬로 정렬해 놓았다고 가정한다면, 어떻게 되나면:
`[작은 숫자들...] (중간값) [큰 숫자들...]`

전체 데이터를 두 그룹으로 쪼개는 것
- 왼쪽 그룹(작은 절반): 이 그룹에서 가장 큰 값이 중간값 후보가 된다 -> 최대 힙(Max Heap)으로 관리
- 오른쪽 그룹(큰 절반): 이 그룹에서 가장 작은 값이 중간값의 또 다른 후보 -> 최소 힙(Min Heap)으로 관리

## 코드에서 디테일

### 규칙 1: 왼쪽부터 채우기(개수 맞추기)

```c++
if (maxHeap.size() <= minHeap.size()) maxHeap.push(t);
else minHeap.push(t);
```
- 항상 두 그룹의 크기를 비슷하게 유지한다.
- 홀수 개일때는 무조건 Max Heap이 1개 더 많게 만든다
- 이렇게 하면 전체 개수가 홀수/짝수 상관없이 무조건 `maxHeap.top`이 중간값이 된다.

### 규칙 2: 왼쪽 최댓값은 오른쪽 최솟값보다 작아야한다

```c++
if (!minHeap.empty() && minHeap.top() < maxHeap.top()) {
  // 스왑(Swap) 로직
}
```
- 규칙 1에 따라 개수만 맞춰서 넣다 보면, 왼쪽 그룹에 들어가야 할 작은 숫자가 오른쪽 그룹에 가거나, 큰 숫자가 왼쪽 그룹에 가는 불상사가 발생할수 있다.
- 그래서 값을 넣은 직후 검사해서, Max Heap의 가장 큰 값(왼쪽 대장)이 Min Heap의 가장 작은 값(오른쪽 막내)보다 크다면?
- 둘의 소속이 잘못되었으니 서로 자리를 바꿔준다!
