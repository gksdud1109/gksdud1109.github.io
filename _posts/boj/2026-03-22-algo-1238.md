[문제 링크](https://www.acmicpc.net/problem/1238)

```c++
#include <bits/stdc++.h>
using namespace std;

const int INF = 0x3f3f3f3f;
int N, M, X;

// 원래 지도 (X -> i, 집에 돌아갈 때 사용)
vector<pair<int, int>> adj[1002];
// 간선이 뒤집힌 지도(i -> X, 파티장 갈 때 사용)
vector<pair<int, int>> rev_adj[1002];

vector<int> dijkstra(int st, const vector<pair<int,int>> graph[]){
  vector<int> d(N+1, INF);
  priority_queue<pair<int, int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;

  d[st] = 0;
  pq.push({0, st});

  while(!pq.empty()){
    auto [w, u] = pq.top(); pq.pop();
    if(d[u] != w) continue;

    for(auto [dw, v] : graph[u]){
      if(d[u]+dw < d[v]){
        d[v] = d[u] + dw;
        pq.push({d[v], v});
      }
    }
  }
  return d;
}

int main(){
  ios::sync_with_stdio(0);
  cin.tie(0);

  cin >> N >> M >> X;

  while(M--){
    int u, v, w;
    cin >> u >> v >> w;
    adj[u].push_back({w, v}); // 정방향 저장
    rev_adj[v].push_back({w, u}); // 역방향 저장
  }

  vector<int> dist_to_home = dijkstra(X, adj);
  vector<int> dist_to_party = dijkstra(X, rev_adj);

  int ans = 0;
  for(int i=1; i<=N; i++){
    if(dist_to_party[i] == INF || dist_to_home[i] == INF) continue;
    ans = max(ans, dist_to_party[i] + dist_to_home[i]);
  }
  cout << ans;
}
```

## 핵심 아이디어: 역방향으로 뒤집어 다익스트라를 2번만 실행

원래의 잘 알려진 풀이는 정방향 그래프에서 모든 학생들에 대해 파티장까지 갔다가 돌아오는 경로를 매번 따로따로 구해서 답을 구한다.

하지만 문제에서의 N, 즉 노드가 1000개가 넘어가면 시간 초과가 날 풀이일 뿐더러, 두 번의 연산만으로 끝낼 풀이를 1000번씩 반복하는 것은 비효율적

### step 1. 정방향 그래프에서 X(도착지)를 출발점으로 돌린다.
- 파티장(X)에서 1번집의 거리, 2번집의 거리 ...
- 이 모든 거리들이(파티장에서 집으로의) 하나의 vector에 전부 기록

### step 2. 역방향 그래프에서 똑같이 X를 출발점으로 돌린다.
- 뒤집힌 세계에서 X에서 1번 집으로 가는 최단 경로는, 원래 세계에서 1번 집에서 X로 오는 최단 경로와 동일하다.
- 모든 학생들이 파티장으로 모이는 길이 또 다른 vector에 한번에 다 기록됨.
- 1000명의 학생의 집을 출발점으로 잡고 X까지 다익스트라를 돌리는 것보다 훨씬 효율적

## 역방향 지도 저장 자체가 양방향이 되버리는게 아닌지?

예시: 1번(집), 2번(환승역), 3번(파티장)
단방향 간선이 아래처럼 주어졌다고 해보자
- `1 -> 2` (거리 10)
- `2 -> 3` (거리 10)
- `3 -> 1` (거리 50)

### 다익스트라 실행
① 집에 올 때 (파티장 3번 ➔ 집 1번)
- 사용할 지도: 진짜 원래 지도(`adj`)
- 다익스트라: 파티장 3번에서 출발
- 지도에서는 3번에서 갈 수 있는 곳은 1번 뿐
- 결과: 돌아올 때 최단거리는 50

② 파티장 갈 때 (집 1번 ➔ 파티장 3번)
- 사용할 지도: 두 번째 역방향 가짜 지도(`rev_adj`)
- 다익스트라: 3번에서 역추적 출발
- 지도에서 3번에서 갈 수 있는 곳은 2번
- 2번에서 갈 수 있는 곳은 1번
- 3번에서 1번까지 총 20, 가짜 지도에서 `3 -> 2 -> 1`은 현실에서 `1 -> 2 -> 3`의 거리와 같음
- 결과: 파티장 갈때 최단 거리는 20

### 핵심
실제로 양방향 그래프로 동작했다면 집에 올 때도 20짜리 경로를 탔을것.
정방향으로 집에 올때의 경로 따로, 역방향 지도로 파티장 갈때의 경로를 따로 계산했기 때문에 가능한 결과다!
