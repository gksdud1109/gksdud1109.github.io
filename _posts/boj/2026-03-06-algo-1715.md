[문제 링크](https://www.acmicpc.net/problem/1715)

```c++
#include <bits/stdc++.h>
using namespace std;

int main(){
  ios::sync_with_stdio(0);
  cin.tie(0);
  priority_queue<int, vector<int>, greater<int>> pq;
  int N; cin >> N;
  while(N--){
    int x; cin >> x;
    pq.push(x);
  }

  int ans = 0;
  while(pq.size() > 1) {
    int a = pq.top(); pq.pop();
    int b = pq.top(); pq.pop();
    ans += a+b;
    pq.push(a+b);
  }
  cout << ans;
}
// 누적비용을 최소화하는 그리디 접근, 허프만 코딩과 같은 원리
```

## 핵심 아이디어: 먼저 합친 묶음은 앞으로 계속 더해저야함

카드 묶음을 합칠 때 발생하는 비용을 수식으로 펼쳐보면 이유가 명확해진다.
예를 들어 카드 묶음이 A, B, C, D 4개가 있다고 해보면

1. `A+B`
2. `(A+B) + C`
3. `(A+B+C) + D`

총 누적 = `(A+B)` + `(A+B+C)` + `(A+B+C+D)` = 3A + 3B + 2C + 1D

- 먼저 더할 수록 누적해서 더해지는 횟수가 늘어남
- 총 누적 비용을 최소화해야하기 때문에 가장 많이 중복해서 더해지는 자리에 가장 작은 숫자를 넣어야 전체 합이 최소가 될것이다!

## Priority Queue를 써야하는 이유
가장 작은 두 개를 고르는 것이라면 처음에 배열을 딱 한번 정렬해서 앞에서부터 더해나가면 되지 않을까 의문이 들었다.

하지만 주의할 점은 **두 묶음을 합쳐서 만들어진 새로운 묶음도 다음 비교 대상에 포함된다**는 것

- 합쳐진 새 묶음이 기존에 있던 다른 묶음보다 클 수도, 작을 수도 있음
- 새로운 값이 계속 추가되는 상황에서도 항상 전체 중 가장 작은 두개를 뽑아내야 한다

배열에 값을 새로 넣고 매번 `sort()`를 하게되면(`O(NlogN)`) 시간 초과, 

> 하지만 최소 힙 기반 우선순위 큐는 값을 넣고 뺄 때마다 내부 트리 재배치를 통해 `O(log N)`만에 가장 작은 값을 맨 위로 올릴 수 있다.
