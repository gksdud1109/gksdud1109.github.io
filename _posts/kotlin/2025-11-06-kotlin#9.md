# 13. 예외 처리

```kotlin
try {
    risky()
} catch (e: Exception) {
    println(e.message)
}
```

## 📘 Java와 비교
- Kotlin에는 checked exception이 없음 <br>
→ `throws IOException` 같은 선언 불필요<br>
→ `try-catch`는 선택적 사용

| Java | Kotlin |
|:---|:---|
|throws IOException 강제 명시 | 예외 선언 불필요 |
|Checked / Unchecked 구분 존재 | 모든 예외 Unchecked 취급 |
|try-catch 필수 | 필요 시에만 사용 |

> 💡 활용 팁
>```kotlin
>val result = runCatching {
>    riskyOperation()
>}.onFailure {
>    logger.error("Error: ${it.message}")
>}.getOrNull()
>```
>→ runCatching, onFailure, getOrNull 등으로
>try-catch보다 간결하고 함수형 스타일로 처리 가능

## 예외 처리 — runCatching, onFailure, getOrNull

### 📍기본형 예시
```kotlin
val result = runCatching {
    // 예외 발생 가능 코드
    val value = riskyOperation()
    "결과: $value"
}.onFailure { e ->
    println("에러 발생: ${e.message}")
}.getOrNull()

println(result)
```

### 📍실제 실행 흐름
1.	riskyOperation() 실행 중 예외가 발생하면
→ onFailure 블록으로 이동 (e.message 출력)
2.	이후 getOrNull()은 null 반환

### 📘 Java 대응
```java
String result = null;
try {
    String value = riskyOperation();
    result = "결과: " + value;
} catch (Exception e) {
    System.out.println("에러 발생: " + e.getMessage());
}
System.out.println(result);
```

> 💡 활용 팁
>```kotlin
>val jwt = runCatching {
>    JwtUtil.generateToken(user)
>}.getOrElse {
>    logger.error("JWT 생성 실패: ${it.message}")
>    null
>}
>```
>→ 예외를 던지지 않고 로깅 후 graceful fallback 가능

# 14. 스코프 함수

| 함수 | 리턴값 | 수신 객체 | 사용 목적 |
|:---|:---|:---|:---|
|apply|객체 자체 | this | 초기화용 |
|let | 블록 결과 | it | null-safe 변환 |
|also | 객체 자체 | it | 로깅, 부수효과 |
|run | 블록 결과 | this | 계산, 일회성 처리 |
|with | 블록 결과 | this | 여러 속성 다루기 |


## 📍예제

```kotlin
val person = Person().apply {
    name = "Alice"
    age = 30
}.also {
    println("Created: ${it.name}")
}
```

## 📍 핵심 이해 포인트
-	공통점: 객체를 받아서 코드 블록 내에서 this/it으로 접근
-	차이점: 리턴 값이 다르다 (객체 or 결과)

## 📘 Java와 비교

|Java | Kotlin |
|:---|:---|
|Builder 패턴 or 메서드 체이닝 필요 | apply로 초기화 한 번에 |
|if (obj != null) { … } | obj?.let { … } |
|로그용 print(obj) 후 리턴 | obj.also { println(it) } |

> 💡 활용 팁
> ```kotlin
> val user = userRepository.findByIdOrNull(id)?.let {
>     it.lastLogin = LocalDateTime.now()
>     it
> } ?: throw NotFoundException("User not found")
> ```

<hr>

## apply — 초기화(Builder 패턴 대체)

```kotlin
val person = Person().apply {
    name = "Alice"
    age = 30
}
```

### 📘 Java 대응
```java
Person person = new Person();
person.setName("Alice");
person.setAge(30);
```
➡ 또는 빌더패턴으로 쓰면
```java
Person person = new PersonBuilder()
        .name("Alice")
        .age(30)
        .build();
```
>💡 요약
>
>apply는 this 컨텍스트로 접근하고 객체 자신을 리턴,
>즉, “빌더 패턴 없이 인스턴스 초기화 + 반환”까지 한 번에함

## let — null-safe 변환 / 체이닝

```kotlin
val name: String? = "Alice"
name?.let { println("Length: ${it.length}") }
```
### 📘 Java 대응
```java
String name = "Alice";
if (name != null) {
    System.out.println("Length: " + name.length());
}
```
>💡 요약
>
>let은 “null 체크 + 안전한 사용”을 동시에 수행.
>삼항연산자나 Optional의 ifPresent()보다 간결.

## also — 부수효과(side-effect), 로깅용

```kotlin
val user = User("Alice").also {
    println("Created user: ${it.name}")
}
```

### 📘 Java 대응
```java
User user = new User("Alice");
System.out.println("Created user: " + user.getName());
```
>💡 요약
>
>also는 객체 그대로 반환하면서, 디버깅, 로깅, 통계 카운트 등
>“객체는 그대로, 중간 처리만” 하는 데에 최적화


## run — 계산/임시 블록 (지역 범위)
```kotlin
val length = "Hello".run {
    println("Calculating length...")
    length
}
```

### 📘 Java 대응
```java
String str = "Hello";
System.out.println("Calculating length...");
int length = str.length();
```

>💡 요약
>
>run은 this로 수신객체를 다루고, 블록 결과를 반환.
>한 객체에 대해 “여러 연산을 묶어서 하나의 값으로 리턴”할 때 유용.

## with — 여러 속성 조작 (동일 객체에 연속 작업)
```kotlin
with(person) {
    name = "Alice"
    age = 30
    println("$name is $age years old")
}
```

### 📘 Java 대응
```java
Person person = new Person();
person.setName("Alice");
person.setAge(30);
System.out.println(person.getName() + " is " + person.getAge() + " years old");
```
>💡 요약
>
>with(obj)는 “obj.” 반복을 줄이는 문법.
>동일 객체에 연속 접근할 때 코드가 깔끔해짐

## 📘 정리

함수	Java 제어 구조 대응	Kotlin 사용 의도

- `apply`:	Builder 패턴, setter 체이닝	초기화 & 반환
- `let`:	null check + if문	안전 호출, 변환
- `also`:	print/log 중간처리	부수효과용
- `run`:	지역 변수 연산	계산 결과 반환
- `with`:	동일 객체 반복 접근	다중 속성 설정

# 15. lateinit & lazy

## 📍lateinit

```kotlin
lateinit var name: String
name = "Alice"
println(name)
```
-	var만 가능, null 허용 X
-	나중에 초기화

### 📘 Java와 비교

| Java | Kotlin |
|`private String name; → 나중에 set`| `lateinit var name: String`|
|null 방어는 직접 해야 함 | 컴파일러가 초기화 여부 추적 |

>💡 활용 팁
>
>-	Spring에서 `@Autowired lateinit var service: MyService`
>-	테스트에서 mock 주입 시 활용


## 📍lazy

```kotlin
val msg: String by lazy {
    println("init")
    "Hello"
}
println(msg)
println(msg)
```

### 📍 동작 순서 설명
1.	val msg 정의 시점에는 초기화 코드가 실행되지 않음
2.	처음 msg를 읽는 순간
  -	lazy 블록이 실행되고 "init" 출력
  -	"Hello"가 평가되어 msg에 저장
3.	두 번째 msg 접근 시
  -	이미 초기화 완료 → 블록 실행 X
  -	캐시된 "Hello" 그대로 반환


### 📘 Java로 표현하면

```java
private String msg;

public String getMsg() {
    if (msg == null) {
        System.out.println("init");
        msg = "Hello";
    }
    return msg;
}
```

> 💡 활용 팁
> -	무거운 객체(DB 연결, 설정 파일 등) 초기화 시점 지연
> -	“로그인 후 생성되는 객체” 등에 적합

# 16. companion object (정적 멤버)

```kotlin
class MathUtil {
    companion object {
        val PI = 3.14
        fun square(n: Int) = n * n
    }
}
println(MathUtil.PI)
println(MathUtil.square(4))

```

## 📘 핵심 개념
-	companion object는 클래스 내부에 선언된 유일한 정적 객체
-	Java의 static과 유사하지만, 객체처럼 다룸
-	클래스 로드 시점에 자동 초기화되며 thread-safe

> 💡 활용 팁
>```kotlin
>class JwtUtil {
>    companion object {
>        fun generate(userId: Long): String = ...
>    }
>}
>JwtUtil.generate(1L)
>```
>→ Spring Bean을 쓰지 않는 유틸리티, 상수, 팩토리 메서드 등에 자주 사용

# 17. open / override (상속 허용)

```kotlin
open class Animal {
    open fun sound() = println("Some sound")
}
class Dog : Animal() {
    override fun sound() = println("Bark")
}
```
## 📘 핵심 개념
-	Kotlin의 클래스와 함수는 기본적으로 final
-	상속과 오버라이드를 허용하려면 명시적으로 open 추가
-	override 시 반드시 override 키워드 필요 (컴파일러 강제)

> 💡 활용 팁
>-	BaseEntity, AbstractService 등 공통 로직 클래스는 반드시 open class로 선언해야 함
>-	Spring JPA Entity 클래스는 open 필요 (프록시 사용을 위해)

# 18. Getter / Setter

`class Person(var name: String)`

→ 자동으로 `getName()` / `setName()` 생성됨

## 📍커스텀 Getter/Setter

```kotlin
class Person(private var _name: String) {
    var name: String
        get() = if (_name.isEmpty()) "noname" else _name
        set(value) {
            _name = if (value.isEmpty()) "noname" else value
        }
}
```

# 19. 생성자

## 📍주 생성자

```kotlin
class Person(val name: String, val age: Int)
```
→ 파라미터를 직접 프로퍼티로 선언 가능
→ val / var 붙이면 자동으로 필드 + getter/setter 생성

## 📍 init 블록

```kotlin
class Person(val name: String) {
    init { println("$name 생성됨") }
}
```
→ 생성 시점에 실행되는 초기화 블록

## 📍 부 생성자 

```kotlin
class Person(val name: String, val age: Int) {
    constructor(name: String): this(name, 0)
}
```
→ 주 생성자 호출 필수 (: this(...))

## 📘 Java 대응
```java
class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
        System.out.println(name + " 생성됨");
    }

    public Person(String name) {
        this(name, 0);
    }
}
```

# 20. 컬렉션 고급 문법

```kotlin
val nums = listOf(1,2,3,4,5,6)
val even = nums.filter { it % 2 == 0 }
val doubled = nums.map { it * 2 }
```

## 📘 Java와 비교

| Java | Kotlin |
|:---|:---|
|Stream 필요 | 컬렉션 기본 제공 |
|`.stream().filter(...).map(...).collect()`| `.filter {...}.map {...}`|
|Optional 병합 불편 | 함수형 조합이 간결 |

> 💡 활용 팁
>```kotlin
>val topUsers = users.filter { it.score > 80 }.sortedByDescending { it.score }
>```
>→ API 응답 전 데이터 후처리 시 패턴

# 22. private val: 읽기 전용

```kotlin
class Person(val name: String)
val p = Person("Alice")
println(p.name)
```
-	val → read-only (getter만 생성)
-	var → read-write (getter + setter 생성)
-	내부적으로 private final + getName() 구조로 컴파일됨
