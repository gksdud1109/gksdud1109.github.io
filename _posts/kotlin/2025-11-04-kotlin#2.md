# 코틀린의 null-safe / 간결함 관련 문법들을 한눈에 정리

아래 표는 ?. 와 ?: (엘비스) 같은 문법과, 그 “패밀리 문법들”까지 전부 정리한 거임👇


## 🌟 Kotlin의 핵심 문법 총정리

| 문법 | 이름 | 의미 / 사용 예시 | 비고 |
|:---|:---|:---|:---|
|?. |Safe Call (세이프 콜) |user?.name → user가 null이면 null 반환 | null 안전하게 프로퍼티 접근 |
|?: | Elvis 연산자 | name ?: "이름없음" → name이 null이면 "이름없음" | null 시 기본값 지정 |
| !! | Non-null 단언| user!!.name → null이면 NPE| 가능한 한 쓰지 말기 |
| ?.let { ... } | 널 아닐 때만 실행 | user?.let { println(it.name) } | null이면 실행 안 함|
|?: return / ?: throw | 엘비스 + 흐름 제어 |val x = user ?: return / ?: throw Error() | null이면 바로 리턴/예외 |
| if (a is Type) | 스마트 캐스트 | if (obj is String) println(obj.length) | 타입 검사 후 자동 캐스팅 |
| as? | 안전한 캐스팅 | val s = obj as? String → 실패 시 null | ClassCastException 방지 |
| runCatching {} |예외 포착 블록 | runCatching { risky() }.getOrNull() | try-catch 대신 |
| let | 컨텍스트 함수 (scope) | x.let { it + 1 } | 블록의 결과를 반환 |
| also | 부수효과용 (logging 등) | x.also { println(it) } |원본 객체 반환 |
| apply | 객체 초기화 | User().apply { name="A" } | 자기 자신을 반환 |
| run | 즉시 실행 블록 | run { val x=1; x+2 } | 지역 컨텍스트에서 결과 반환 |
| with(obj) | 수신객체 단축 | with(user){ println(name) } |수신객체 반복 제거 |
| takeIf {} | 조건이 true면 객체 유지 | val adult = person.takeIf { it.age >= 20 } | false면 null 반환
|takeUnless {} | 조건이 false면 객체 유지 | val minor = person.takeUnless { it.age >= 20 }| true면 null 반환 |
| when | switch 대체 | when(x){ 1->"one"; else->"etc" } |표현식이라 값 반환 가능 |
| in / !in | 포함 검사 | if (x in list) | contains() 호출 대체 |
| is / !is | 타입 검사 | if (obj is String) | instanceof 대체
| :: | 참조 연산자 | ::println / User::class | 함수, 프로퍼티, 클래스 참조 |
| ?.::method | nullable 함수 참조 | obj?.::doWork | 안전한 함수 참조 |
| val a by lazy { ... } | 지연 초기화 | 처음 접근 시에만 블록 실행 | 불변 lazy 값|
|lateinit var | 지연 주입 변수 | DI, 테스트용 등 | null 아님 단언해야|
|ifExpression | if 자체가 값| val x = if(a>0) 1 else -1 | 자바엔 없음|
|tryExpression |try도 값 | val x = try { risky() } catch(e:..) { null } | try 결과 반환 |


## 🔍 Null 처리 / 안전 호출 관련 문법만 따로 보기

| 패턴 | 예시 | 설명 |
|:----|:----|:----|
| 안전 호출 |a?.b | a가 null이면 null 반환 |
| 엘비스 기본값 | a ?: default | null이면 default 반환 |
|안전 실행 | a?.let { ... } | null 아닐 때만 실행 |
|안전 캐스팅 | a as? Type | 타입 안 맞으면 null |
|조건 유지 | a.takeIf { it > 0 } | 조건 false면 null|
|조건 반전 유지 | a.takeUnless { it > 0 } | 조건 true면 null
|null 강제 제거 | a!! | null이면 NPE 발생 |
|널 체크 함수 | isNullOrEmpty(), isNullOrBlank() | null과 빈 문자열을 동시에 검사 |


## 💎 스코프 함수들 정리 (let / also / run / apply / with)

|함수 | 반환값 | 주 용도 | 코드 예시 |
|:---|:---|:---|:---|
|let | 블록의 결과값 | 변환 / 체이닝 | x?.let { it + 1 } |
|also | 자기 자신 | 부수효과 (로그) | x.also { println(it) } |
| run | 블록의 결과값 | 임시 변수 없이 코드 묶기 | run { val y=1; y+2 } |
|apply | 자기 자신 | 객체 초기화 | User().apply { name="A" } |
| with | 블록의 결과값 | 수신객체 지정 블록 | with(user){ name.uppercase() } |

## 🚀 실전 예시

```kotlin
val redirectUrl = request.getParameter("state")
    ?.takeIf { it.isNotBlank() }                // null 또는 공백이면 중단
    ?.let { Base64.getUrlDecoder().decode(it) } // null 아니면 decode
    ?.toString(Charsets.UTF_8)
    ?.split("#")
    ?.getOrNull(1)
    ?: "/"                                     // 모든 게 실패하면 기본값
```

# 🎯 요약
-	코틀린은 “null을 코드로 관리하는 언어”다.
-	?., ?:, let, run, apply, takeIf, as? 이 여섯 가지만 익히면
자바의 if/try-catch 10줄이 코틀린에선 1줄로 가능.
-	모든 표현식이 값을 반환할 수 있다는 점(if, when, try)도 자바와의 큰 차이.
