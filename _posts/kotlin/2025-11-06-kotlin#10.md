# 🌱 23. Spring 실무 적용 핵심 패턴

| 패턴 | Kotlin 문법 | 실무 활용 / Java 대비 포인트 |
|:---|:---|:---|
| **DI (의존성 주입)** | `class Service(private val repo: Repo)` | 생성자 주입 기본, `@Autowired` 생략 가능 (`constructor` 자동 인식) |
| **Entity 정의** | `@Entity open class Post(... var title: String)` | JPA 프록시 위해 `open class`, `open var` 필요 (Bytecode Proxy) |
| **DTO 정의** | `data class PostDto(val id:Long, val title:String)` | JSON 직렬화, equals/toString 자동 생성 |
| **Repository 확장** | `fun findByIdOrNull(id: Long)` | Kotlin Spring 확장 함수 (`Optional` 대체) |
| **Service 예외 처리** | `throw ServiceException("404", "Not Found")` | 커스텀 예외로 상태코드·메시지 관리 |
| **Controller 반환 구조** | `fun create() = RsData("200", "OK", data)` | `ResponseEntity` 대체 → 통일된 응답 래퍼 사용 |
| **Exception 정의** | `class MyException(msg:String):RuntimeException(msg)` | 간결한 Custom Exception |
| **Builder 패턴 대체** | `.apply { name = "Alice"; age = 30 }` | 객체 초기화 시 Setter 체이닝 대체 |
| **유틸리티 클래스** | `object FileUtil { fun copy() {...} }` | Java static 유틸 클래스 대신 `object` 사용 |
| **로깅/디버깅** | `.also { logger.info("Created: $it") }` | 체이닝 중간 로깅, side-effect 로직 삽입 |
| **Null-safe 조회** | `repo.findByIdOrNull(id) ?: throw NotFound()` | `Optional.orElseThrow()` 보다 간결 |
| **Bean 등록 불필요** | `@Service / @Component` + Kotlin 클래스 | Kotlin 기본 생성자 주입으로 boilerplate 제거 |
| **RsData 패턴** | `data class RsData(val code:String, val msg:String, val data:T?)` | Controller 응답 표준화, 일관된 구조 유지 |
| **테스트 주입용 필드** | `@Mock lateinit var repo: Repo` | `lateinit`을 이용해 나중에 초기화 가능 |
| **Config / Const 관리** | `object AppConfig { const val VERSION = "1.0" }` | 상수 + 설정 관리 (static field 대체) |
| **Mapper 변환** | `fun Member.toDto() = MemberDto(id, name)` | Entity→DTO 변환 확장함수로 관리 |



# ⚙️ 최종 요약 - kotlin 핵심 한눈에 보기

| 개념 | 핵심 문법 | 실무 포인트 |
|:---|:---|:---|
| **변수 선언** | `val`, `var` | `val` 기본 (불변성 우선) |
| **문자열 템플릿** | `"Hello $name"` | `${expression}` 사용 가능 |
| **타입 추론** | `val x = 10` | 타입 생략 가능, 명시 시 `: Int` |
| **Null 처리** | `?`, `?.`, `?:`, `!!` | null-safe 문법 (`?:` 기본값 제공) |
| **조건문** | `if`, `when` | if는 표현식, when은 switch 대체 |
| **반복문** | `for (i in 0..n)` | `..` 포함, `until` 미포함 |
| **함수 선언** | `fun add(a:Int,b:Int)=a+b` | 반환 타입 생략 가능, 단일 표현식 함수 |
| **기본 매개변수** | `fun greet(name:String="Guest")` | 오버로딩 불필요 |
| **람다 / 고차함수** | `list.map { it*2 }` | Stream API보다 간결 |
| **데이터 클래스** | `data class User(val id:Long, val name:String)` | equals, hashCode, toString 자동 생성 |
| **확장 함수** | `fun String.greet() = println("Hi $this")` | 유틸 클래스 대체, 타입 확장 |
| **싱글톤** | `object Logger { fun log(msg:String){...} }` | 전역 유일 객체, static 대체 |
| **정적 멤버** | `companion object` | Java static 멤버 대체 |
| **예외 처리** | `try/catch`, `runCatching` | checked 예외 없음, 함수형 예외 처리 |
| **스코프 함수** | `apply`, `let`, `also`, `run`, `with` | 객체 초기화, 변환, 로깅 등에 활용 |
| **상속 제어** | `open`, `override` | 기본 final, 명시적 open 필요 |
| **Getter/Setter** | `var name:String` | Lombok 불필요, 자동 생성 가능 |
| **생성자 / init** | `class Person(val n:String){init{...}}` | 생성자와 초기화 동시 관리 |
| **lazy 초기화** | `val token by lazy{ loadToken() }` | 최초 접근 시 1회 실행 |
| **lateinit** | `lateinit var repo: Repo` | DI / 테스트 시 지연 주입 |
| **타입 체크** | `is`, `as?` | Smart Cast로 자동 캐스팅 |
| **컬렉션 처리** | `filter/map/sortedBy` | Stream보다 직관적 |
| **private val** | `val name:String` | getter만 생성 (읽기 전용) |
| **DI** | `constructor(private val repo: Repo)` | `@Autowired` 생략, 생성자 자동 주입 |
| **Entity(JPA)** | `open class`, `open var` | 프록시 안전성 확보 필수 |
| **DTO/Response** | `data class` | JSON 응답 직렬화 기본 지원 |
| **Controller 응답** | `fun create() = RsData("200","OK")` | ResponseEntity 대체 |
| **Custom Exception** | `class NotFound:RuntimeException("404")` | 단일 라인 정의 가능 |
| **Builder 대체** | `.apply { title="Hello" }` | 객체 초기화 간결화 |
| **유틸 클래스** | `object Util { fun now()=... }` | static 유틸 대체 |
| **로깅** | `.also { println(it) }` | 체이닝 중간 로그 출력 |
| **Null-safe 조회(JPA)** | `repo.findByIdOrNull(id) ?: throw ...` | Optional 대체 |
| **표준 응답 구조** | `data class RsData(val code:String, val msg:String, val data:T?)` | Controller 응답 통합 관리 |
