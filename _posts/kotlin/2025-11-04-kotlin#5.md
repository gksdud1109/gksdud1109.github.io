# GlobalExceptionHandler 리팩터링 코드

```kotlin
@ControllerAdvice
class GlobalExceptionHandler {
    @ExceptionHandler(NoSuchElementException::class)
    @ResponseBody
    fun handleException(e: NoSuchElementException?): RsData<Void?> =
        RsData("404-1","존재하지 않는 데이터입니다.")

    @ExceptionHandler(MethodArgumentNotValidException::class)
    @ResponseBody
    fun handleException(e: MethodArgumentNotValidException): RsData<Void?> {
        val message = e.bindingResult.allErrors
            .filterIsInstance<FieldError>()
            .sortedWith(compareBy({ it.field }, { it.code }))
            .joinToString("\n") { "${it.field}-${it.code}-${it.defaultMessage}"}

        return RsData("400-1", message)
    }

    @ExceptionHandler(HttpMessageNotReadableException::class)
    @ResponseBody
    fun handleException(e: HttpMessageNotReadableException?): RsData<Void?> =
        RsData<Void?>("400-2", "잘못된 형식의 요청 데이터입니다.")

    @ExceptionHandler(ServiceException::class)
    @ResponseBody
    fun handleException(e: ServiceException): RsData<Void?> =
        RsData(e.getResultCode(), e.getMsg())
}
```

# 주요 포인트


## 원래 Java 코드 버전

```java
@ExceptionHandler(MethodArgumentNotValidException.class)
@ResponseBody
public RsData<Void> handleException(MethodArgumentNotValidException e) {
    String message = e.getBindingResult()
        .getAllErrors()
        .stream()
        .filter(error -> error instanceof FieldError)
        .map(error -> (FieldError) error)
        .map(error -> error.getField() + "-" + error.getCode() + "-" + error.getDefaultMessage())
        .sorted(Comparator.comparing(String::toString))
        .collect(Collectors.joining("\n"));

    return new RsData<>("400-1", message);
}
```

## 💬 이 코드의 동작 순서

1.	getAllErrors() → 모든 ObjectError 목록을 가져옴.<br>
(여기엔 필드 검증 오류(FieldError)와 글로벌 오류가 섞여 있음.)
2.	.filter(error -> error instanceof FieldError) <br>
→ 필드 단위 오류만 남김.
3.	.map(error -> (FieldError) error)<br>
→ 형변환해서 필드 이름, 코드, 메시지에 접근 가능하게 만듦.
4.	.map(error -> "필드명-코드-메시지")<br>
→ 문자열 형태로 변환.
5.	.sorted(Comparator.comparing(String::toString))<br>
→ 정렬 (예: title-NotBlank → title-Size 순)
6.	.collect(Collectors.joining("\n"))<br>
→ 여러 줄의 문자열로 합침.

## Kotlin 버전이 하는 일 — 완전히 동일한 흐름

```kotlin
val message = e.bindingResult.allErrors
    .filterIsInstance<FieldError>()
    .sortedWith(compareBy({ it.field }, { it.code }))
    .joinToString("\n") { "${it.field}-${it.code}-${it.defaultMessage}" }
```

| Kotlin 코드 | Java Stream 코드 | 역할 |
|:----|:----|:----|
|e.bindingResult.allErrors | e.getBindingResult().getAllErrors() | 모든 오류 리스트 가져오기 |
|.filterIsInstance<FieldError>() | .filter(error -> error instanceof FieldError) | FieldError만 필터링 |
|.sortedWith(compareBy({ it.field }, { it.code })) | .sorted(Comparator.comparing(String::toString)) | 필드명 → 코드 순으로 정렬 |
|.joinToString("\n") { "${it.field}-${it.code}-${it.defaultMessage}" } | .map(error -> "field-code-message").collect(Collectors.joining("\n"))| 문자열로 합쳐서 반환 |


## 🔍 세부 문법 차이 해설

### `filterIsInstance<FieldError>()`
> Java: `.filter(error -> error instanceof FieldError).map(error -> (FieldError) error)`

Kotlin은 instanceof + 캐스팅을 동시에 처리하는 편의 함수가 있음.
즉, FieldError 타입인 요소만 남기고 자동으로 캐스팅까지 수행해 줌.

그래서 filterIsInstance<FieldError>() 한 줄이면 .filter + .map + cast 세 줄을 대체.

### `compareBy({ it.field }, { it.code })`
> Java: `.sorted(Comparator.comparing(String::toString))`

Kotlin에서는 여러 필드를 기준으로 정렬할 때 compareBy를 쓴다.
여기선
1. 먼저 field (필드명)
2. 그다음 code (검증 어노테이션 이름)

순으로 정렬 기준을 준 거임

즉,
title-NotBlank
title-Size
이런 순서가 보장되도록 정렬

### `joinToString("\n") { "${it.field}-${it.code}-${it.defaultMessage}" }`
> Java: `.map(error -> "field-code-message").collect(Collectors.joining("\n"))`

Kotlin의 joinToString()은 Collectors.joining()의 단축 버전.
map + join을 한 번에 처리할 수 있음

-	`${it.field}-${it.code}-${it.defaultMessage}` 는 문자열 템플릿.
(Java의 "field-" + error.getCode() + "-" + ... 와 같음.)
-	"\n" 은 줄바꿈 구분자.
