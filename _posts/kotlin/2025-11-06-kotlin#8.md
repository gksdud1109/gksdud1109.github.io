# Kotlin 핵심 문법 정리

# 1. 기본 문법

| 구분 | Java | Kotlin | 특징 |
|----|----|----|----|
|변수 선언 | int a = 10; | val a = 10 / var a = 10 |val: 불변, var: 가변 |
|출력 | System.out.println() | println() | 간결한 기본 출력 |
|문자열 | + 연산자 연결 | "Hello $name" | 문자열 템플릿 |
| 타입 추론 | 명시적 | 자동 추론 | 타입 생략 가능 |
|세미콜론 | 필수 | 생략 | 문장 끝에 불필요 |


## 📍val/var 차이

```kotlin
val x = 10     // 변경 불가
var y = 20     // 변경 가능
```

📘 핵심 개념 정리
-	Kotlin은 “불변이 기본, 가변은 예외” 철학을 따름<br>
→ 자바의 final보다 더 넓게 적용되는 개념.<br>
→ 멀티스레드 안전성 / 사이드 이펙트 최소화에 기여.
-	타입 추론은 동적 언어처럼 보여도 실제로는 정적 타입 언어(compile-time check).
즉, “타입을 생략할 뿐, 타입 안정성은 그대로 유지됨”.

# 2. 제어문

## 📍if / else

→ Kotlin의 if는 표현식 (값 반환 가능)

```kotlin
val num = 10
val str = if (num % 2 == 0) "Even" else "Odd"
```

## 📘 Java 대비 포인트
-	Kotlin의 if는 **statement(문)** 이 아니라 expression(값).
즉, 삼항 연산자 ? :가 필요 없음.
-	return if(...)처럼 직접 반환 가능 → 함수형 스타일로 자연스럽게 연결 가능.

```kotlin
fun grade(score: Int) = if (score >= 90) "A" else "B"
```

## 📍when (switch 대체)

→ Java 14의 switch 표현식보다 간결

```kotlin
val day = 3
val name = when (day) {
    1 -> "Mon"
    2 -> "Tue"
    3 -> "Wed"
    else -> "Invalid"
}
```

## 📘 Java 대비 포인트
-	break 문이 필요 없음 (각 분기가 표현식)
-	값 뿐 아니라 타입 매칭도 가능
-	여러 값 묶기 가능

```kotlin
// 타입 매칭
when (obj) {
    is String -> println("문자열")
    is Int -> println("정수")
    else -> println("기타 타입")
}
// 여러 값 묶기
when (x) {
    0, 1 -> println("0 또는 1")
}
```

# 3. 반복문

```kotlin
for (i in 0 until 5) println(i)   // 0~4
for (i in 0..4) println(i)        // 0~4 (동일)
for (i in 5 downTo 1 step 2) println(i) // 5,3,1
```

## 📘 Java 대비 포인트
-	i++, i < n 형태의 전통 for 문 없음
-	Range (IntRange) 기반 반복 구조<br>
→ 0..5, 0 until 5, 5 downTo 1, step 2 등 다양한 조합 가능
-	컬렉션 순회는 for (item in list)로 통일됨
-	Iterator나 index 기반 반복도 indices, withIndex()로 처리:

```kotlin
for ((index, value) in list.withIndex()) println("$index: $value")
```

# 4. 함수

```kotlin
fun add(a: Int, b: Int): Int = a + b
```
## 📘 Java 대비 포인트
-	fun 키워드로 함수 정의
-	반환 타입은 맨 뒤에 명시 (Int)
-	한 줄짜리 함수는 =로 표현식 형태로 작성 가능
-	반환형 추론 가능 → fun add(a: Int, b: Int) = a + b 도 가능

## 📍기본값 매개변수 + 이름 지정 인자

```kotlin
fun greet(name: String = "Guest", age: Int = 20) = println("$name, $age")
greet(age = 25)  // 이름 지정 호출
```
→ 자바의 오버로딩 필요 없음

# 5. 클래스 & 객체 (객체지향 간소화)

```kotlin
class Person(val name: String) {
    fun greet() = println("Hello, $name")
}
val p = Person("Alice")
p.greet()
```
## 📘 Java 대비 포인트
-	생성자 매개변수에 val or var → 자동으로 필드 + getter 생성<br>
→ val name: String = private final String name; + getName()
-	생성자 코드는 init 블록에서 추가 가능
-	new 키워드 없음 (Person() 바로 호출)
-	final이 기본 → 상속하려면 open 필요

```kotlin
open class Animal(val name: String) {
    open fun sound() = println("...")
}
class Dog(name: String) : Animal(name) {
    override fun sound() = println("멍멍")
}
```

# 6. 컬렉션


## 📍java 코드

```java
List<String> list = new ArrayList<>();

Map<String, Integer> map = new HashMap<>();
```

## 📍kotlin 코드

```kotlin
val list = mutableListOf("A", "B")

val map = mutableMapOf("A" to 1, "B" to 2)
```

## 📘 Kotlin 컬렉션의 핵심 설계 개념
-	Kotlin에서는 불변과 가변이 분리됨
-	listOf() → 읽기 전용 (immutable)
-	mutableListOf() → 추가/삭제 가능
-	Map도 동일하게 mapOf() / mutableMapOf() 로 구분

활용 예:

```kotlin
val nums = listOf(1, 2, 3, 4, 5)
nums.filter { it % 2 == 0 }
    .map { it * 2 }
    .forEach(::println)
```
## 📘 Java와 비교

java stream
`numbers.stream().filter(...).map(...).collect(Collectors.toList())`
kotlin
`numbers.filter {...}.map {...}`

-	Kotlin의 컬렉션은 내부적으로 Stream API와 유사한 고차함수 지원
-	람다 사용이 기본 전제라, Java보다 훨씬 간결함
-	`::println`은 함수 참조 (메서드 레퍼런스)

>💡 활용 팁
>
>-	백엔드 데이터 가공, 응답 DTO 변환 시 filter/map/associate 등을 적극 활용
>-	.also나 .onEach를 함께 써서 중간 디버깅도 자연스럽게 가능

# 7. Null-Safety

| Java | Kotlin |
|:----|:----|
| if (name != null) name.length() | name?.length |
| 삼항연산자 (?:) | name ?: "Guest" |
| Optional.ofNullable(name).orElse("Guest") | 내장 문법으로 해결 |

## 📍 예시
```kotlin
val name: String? = null
println(name?.length ?: "Name is null")
```

## 📘 Kotlin이 Null을 다루는 방식
-	String? → null 가능 타입
-	String → null 불가능 타입
-	?. → 안전 호출 (null이면 그냥 null 리턴)
-	?: → 엘비스 연산자 (null이면 기본값 사용)
-	!! → 강제 non-null (NPE 발생 위험)

## 📘 Java와의 구조적 차이

| 항목 | Java | Kotlin |
|:---|:---|:---|
| Null Pointer 방어 | if, Optional | 언어 레벨에서 타입 시스템이 방어 |
|Null 발생 시 | 런타임 에러 | 컴파일러가 강제 검사 |
|NPE 가능 변수 | 모든 변수 | ? 붙은 변수만 |

>💡 활용 팁
>```kotlin
>val user = userRepository.findByIdOrNull(id) 
>    ?: throw ServiceException("404", "유저를 찾을 수 없습니다.")
>```

# 8. 람다 & 고차 함수

```kotlin
val names = listOf("Alice", "Bob", "Charlie")
names.forEach { println(it) }
```
-	`it` → 단일 매개변수 이름 생략 가능
-	`map, filter, forEach, reduce` 등 고차함수 내장

## 📘 기본 문법
-	`{ ... }`→ 람다 블록
-	`it` → 단일 매개변수의 기본 이름
-	`(T) -> R` → 함수 타입 (ex: `(Int) -> String`)

## 📘 Java와 비교

|Java | Kotlin |
|:---|:---|
|`names.forEach(name -> System.out.println(name));` | `names.forEach { println(it) }` |
| `(x, y) -> x + y` | `{ x, y -> x + y }` |
| Stream 필요 | 리스트 기본 지원 |


## 💡 고차 함수란? : “함수를 인자로 받거나, 함수를 반환하는 함수”

```kotlin
fun operate(x: Int, op: (Int) -> Int): Int = op(x)

fun main() {
    println(operate(5) { it * 2 }) // 10
}
```

# 9. 데이터 클래스 (data class)

```kotlin
data class Person(val name: String, val age: Int)
```

자동 생성:
-	equals, hashCode, toString, copy, componentN()

```kotlin
val p1 = Person("Alice", 25)
val p2 = p1.copy(age = 30)
println(p1 == p2) // false
```

## 📘 Java와 비교

| Java | Kotlin |
|:---|:---|
|`record Person(String name, int age) {}`| `data class Person(val name: String, val age: Int)` |
|Getter/Setter/equals/toString 수동 작성 (or lombok) | 자동 생성 |

>💡 활용 팁
>
>```kotlin
>data class PostDto(val id: Long, val title: String, val content: String)
>```
>→ JPA Entity와 분리된 응답용 DTO로 활용

# 10. 확장 함수

```kotlin
fun String.greet() = println("Hello, $this!")
"Bob".greet()
```

```kotlin
fun List<Int>.square() = map { it * it }
println(listOf(1,2,3).square())
```
→ 기존 클래스 수정 없이 기능 확장 가능

## 📘 원리
-	컴파일 시 static 함수로 변환되지만, 호출은 인스턴스 메서드처럼 사용 가능.
-	확장 함수는 기존 코드 수정 없이 기능 확장 가능.
-	실제로 this 수신 객체를 넘겨받는 syntactic sugar.

>💡 활용 팁
>
>```kotlin
>fun HttpServletRequest.getUserAgent(): String =
>    this.getHeader("User-Agent") ?: "Unknown"
>```

# 11. 기본 매개변수 & 오버로딩 제거

```kotlin
fun sayHello(name: String = "Guest") {
    println("Hello, $name")
}

sayHello() // "Hello, Guest"
sayHello("Alice") // "Hello, Alice"
```
## 📘 핵심 개념
- Kotlin에서는 함수 매개변수에 기본값(default value) 을 지정할 수 있음.
- 오버로딩(overloading)을 대체하는 방식으로 설계됨.

→ sayHello()와 sayHello("Alice") 모두 동일 함수로 처리.

## 📘 Java와 비교

| Java | Kotlin |
|:---|:---|
|여러 오버로딩 메서드 작성 필요 | 한 함수에서 기본값 지정 |
|`void greet() + void greet(String name)`| `fun greet(name: String = "Guest")`|
|컴파일러가 시그니처별 구분 | 호출 시 기본값 적용 |

> 💡 활용 팁
>```kotlin
>fun createUser(username: String, role: String = "USER") { ... }
>```
>→ API 서비스 계층에서 기본값 파라미터 설정 시 매우 유용<br>
>→ 자바에서 빌더패턴이나 오버로딩으로 풀던 걸 한 줄로 정리 가능

# 12. Singleton (object)


```kotlin
object Singleton {
    fun show() = println("Hello from Singleton!")
}
Singleton.show()
```
## 📘 핵심 개념
-	Kotlin의 object 키워드는 전역 유일 인스턴스를 생성함.
-	JVM 레벨에서 thread-safe하게 단 한 번만 생성됨.

## 📘 Java와 비교

| Java | Kotlin |
|:---|:---|
|`private static final Singleton INSTANCE = new Singleton();` | `object Singleton`|
|생성자 private + getInstance() 필요| 자동 인스턴스 관리 |
|Lazy 초기화 처리 별도 구현 필요 | 기본적으로 Lazy & Thread-safe |

>💡 활용 팁
>
>```kotlin
>object JwtUtil {
>    fun generateToken(): String = ...
>}
>```
>→ 유틸리티, 설정 객체, 공용 Bean 대체용으로 활용. <br>
>→ 자바 static 메서드보다 테스트/DI 친화적.
